Coding Standards

In order to keep the source code sane, a standard has to be followed in the coding practice.

Note

These standards are ripped directly from the Group Time project and are subject to change. Soon I will update this with info on how to interface code into Galore and change a few of these noted practices.
  • Files must be saved in Unix format and for indentation should be used 2 spaces (not TABs)
  • The start and end tags for PHP script must be: <? ?>
  • Define necessary constants before they are being called (ie, in include files)
  • All files, functions and classes should be commented so properly that even the developer, who is not familiar with the code, can easily understand.
  • All variables should be called within their superglobals format ($_POST, $_GET, $_SESSIONS, $_SERVER etc).
  • Use include(), include_once(), require() and require_once() where appropriate:
    require('file.php');

    if (condition) {
      include('file_true.php');
    } else {
      doSomething();
    }
  • Instances of objects should be called as:
    new ClassName;
    and not:
    new ClassName();
    When parameters are passed, then use:
    new ClassName($parameter1);
  • To echo out a string, use:
    echo $variable;
    and not:
    print $variable;
    <?=$variable;?>
  • All the custom function names must begin with lowercase word and if theres another word, must have the first letter capitalized:
    function myFunction($parameter, $optional = '') {
      global $HTTP_GET_VARS, $another_variable;
      <function code>
      return true;
    }
  • Class names should match its lower-cased file-based name: myclass.php. Classnames should be declared with the words first letters capitalized. The method names must begin with lowercase word and if theres another word, must have the first letter capitalized ( (get(), doSomething(), getName(), etc.)
    class MyClassName {
      var $variable;
      // class constuctor
      function myClassName() {
        <method code>
        return true;
      }

      // class methods
      function doSomething() {
        $this->variable = 'set';
        return true;
      }
    }

    $class = new MyClass;
    $class->doSomething()
  • Database queries should be done in this format, where all the SQL keywords are uppercases:
    $actionQuery = dbQuery("SELECT column1, ... FROM table WHERE ...");
    while ($action = dbFetchArray($actionQuery)) {
      echo $action['column1'];
    }
  • All functions should return strings and not do any direct echoing, for example:
    function myFunction($string) {
      return $string;
    }
    and not:
    function myFunction($string) {
      echo $string;
    }
  • Simple boolean expressions can be written as:
    $string = ($condition) ? 'true' : 'false';
  • IF statements with more than 3 conditions should use SWITCH/CASE:
    switch ($string) {
      case '1':
        doSomething('1');
        break;
      case '2':
        doSomething('2');
        break;
      case '3':
        doSomething('3');
        break;
      default:
        doSomething('default');
        break;
    }
